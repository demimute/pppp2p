# NEXT_STEP

## 当前目标
打通 Electron -> Renderer -> Backend 的一轮可见联调，并定位首个真实交互缺陷。

## 输入条件
- `npm install` 已完成
- 后端已有独立可用实例
- 项目目录结构已存在

## 预计动作
1. 启动并保持 `npm run dev:vite`
2. 启动 `npm run dev:electron`，确认窗口与后端状态链路
3. 检查关键交互路径：选目录、扫描、分析、对比、移除、撤销
4. 记录首个真实缺陷并优先修复

## 验证方法
- `npm run dev:vite` 正常启动并保持运行
- `npm run dev:electron` 可持续运行且不报主进程错误
- 至少确认一条 UI -> API 的交互链路结果，并将缺陷记录到 `ISSUES.md`

## 完成条件
- 明确当前真实阻塞点
- 如果可快速修复，则完成修复并重新验证
- 把下一次 10 分钟进展汇报节点写入执行节奏
