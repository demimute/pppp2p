# DedupStudio - 智能照片去重助手

## 1. 产品概述

**产品名称**: DedupStudio  
**类型**: 跨平台桌面应用（Mac + Windows）  
**核心功能**: 基于 CLIP 视觉嵌入 + 感知哈希的智能照片去重工具，支持多策略、阈值可视化、人工介入确认  
**目标用户**: 普通用户（摄影师、家庭用户）、需要整理大量照片的人群

## 3. 功能规格

### 3.1 文件夹选择

- 用户点击"选择文件夹"按钮，弹出系统原生文件夹选择对话框
- 支持拖拽文件夹到应用窗口
- 显示已选文件夹路径和其中图片数量
- 路径持久化，下次打开自动填充

### 3.2 策略选择

提供 4 种策略，用户可任选其一或组合：

**策略 A: CLIP 视觉相似**
- 使用 CLIP ViT-B/32 模型提取图片 embedding
- cosine similarity 作为相似度度量
- 阈值范围: 0.80 ~ 0.99（步进 0.01）
- 聚类方法: 宽松阈值(0.85)找连通分量 → winner 直接对比严格阈值

**策略 B: 感知哈希 (pHash)**
- 使用 pHash 算法生成 64-bit 哈希
- Hamming 距离作为相似度度量
- 阈值: 距离 <= 10 为重复

**策略 C: 文件大小完全相同**
- 按文件大小（KB）分组
- 同大小文件列为疑似重复

**策略 D: CLIP + pHash 双保险**
- 同时满足 CLIP >= 0.92 且 pHash 距离 <= 10
- 最严格，只找真正重复

### 3.3 阈值滑块

- 滑块拖动时实时显示：
  - 当前阈值
  - 预估移除张数
  - 预估保留张数
  - 分组数量
- 不触发后端重新计算（用预计算数据在前端估算）

### 3.4 相似组网格展示

- 按相似组展示缩略图网格
- 每个组的 winner（最大文件）用绿色边框标注
- 每张缩略图下方显示与 winner 的相似度分数
- 点击单张缩略图 → 弹出侧边对比面板
- 支持键盘导航（↑↓←→）

### 3.5 对比视图（侧边面板）

- 左图：当前选中的照片
- 右图：该组的 winner 照片
- 显示：文件名、文件大小、分辨率、相似度分数
- 操作按钮：[标记保留] [标记移除] [跳过]
- 支持左右箭头键切换当前组内照片

### 3.6 执行与撤销

**执行流程**:
1. 用户点击"执行移除"
2. 弹出确认面板（缩略图网格预览将要移除的照片）
3. 用户确认 → 文件移动到 "照片-已去重" 子目录
4. 操作记录写入历史

**撤销**:
- 底部固定操作历史栏
- 每次操作显示：时间、策略、移除数量
- 点击"撤销"→ 文件移回原目录
- 撤销后 redo 不支持（防止状态混乱）

### 3.7 缓存机制

- CLIP embedding 计算结果缓存到 `~/.dedup-studio/cache/` 目录
- 同一文件夹同一图片，第二次扫描直接读缓存
- 缓存文件命名: `{md5(文件夹路径)}_{图片名}.json`
- 提供"清除缓存"按钮

### 3.8 智能推荐

- 根据照片数量和分布，自动推荐合理阈值
- 分析相似度分布曲线，找到拐点
- 在阈值滑块附近显示推荐标记

## 5. API 接口规范

### POST /api/scan
扫描文件夹，返回图片列表
```json
Request:  { "folder": "/path/to/photos" }
Response: {
  "folder": "/path/to/photos",
  "total": 178,
  "cached": 45,
  "images": [
    { "name": "P1059565.jpg", "size": 342000, "cached": true },
    ...
  ]
}
```

### POST /api/embed
计算 CLIP embedding（分批，每批 10 张）
```json
Request:  { "folder": "/path", "images": ["a.jpg", "b.jpg"] }
Response: { "embeddings": { "a.jpg": [...512 dims...], ... } }
```

### POST /api/hash
计算感知哈希
```json
Request:  { "folder": "/path", "images": ["a.jpg", "b.jpg"] }
Response: { "hashes": { "a.jpg": "1b3b39e4c4a8d0f0", ... } }
```

### POST /api/groups
计算分组结果
```json
Request:  {
  "folder": "/path",
  "strategy": "clip",
  "threshold": 0.93,
  "loose_threshold": 0.85
}
Response: {
  "groups": [
    {
      "id": 1,
      "winner": "P1059619.jpg",
      "winner_size": 775168,
      "members": [
        { "name": "P1059619.jpg", "similarity": 1.0, "to_remove": false },
        { "name": "P1059620.jpg", "similarity": 0.958, "to_remove": true }
      ]
    },
    ...
  ],
  "stats": { "total_groups": 11, "to_remove": 20, "to_keep": 158 }
}
```

### POST /api/move
执行移动
```json
Request:  { "folder": "/path", "moves": [{"name": "a.jpg", "action": "remove"}] }
Response: { "success": true, "moved": 20 }
```

### POST /api/undo
撤销最近操作
```json
Request:  { "folder": "/path" }
Response: { "success": true, "restored": 20 }
```

### GET /api/history
获取操作历史
```json
Response: {
  "history": [
    { "id": 1, "time": "2026-04-17T07:30:00", "strategy": "clip", "threshold": 0.88, "removed": 23 },
    ...
  ]
}
```

## 7. 验收标准

### 功能验收

- [ ] 能选择文件夹并正确扫描图片
- [ ] CLIP 策略能正确计算 embedding 并找相似组
- [ ] 感知哈希策略能正确计算 pHash 并找相似组
- [ ] 阈值滑块拖动时统计数字实时更新
- [ ] 对比视图能正确显示双图和分数
- [ ] 执行移除后文件正确移动到目标目录
- [ ] 撤销能将文件移回原目录
- [ ] 缓存机制正常工作（第二次扫描跳过已计算图片）
- [ ] 深色模式正常切换

### 性能验收

- CLIP embedding: Mac M4 16GB 下，178 张图 <= 3 分钟
- pHash: 178 张图 <= 30 秒
- 相似度计算（178 张）: <= 5 秒
- UI 交互: 滑块拖动无卡顿

### 打包验收

- [ ] Mac: 生成 .dmg 安装包，安装后能正常运行
- [ ] Windows: 生成 .exe 安装包，安装后能正常运行
