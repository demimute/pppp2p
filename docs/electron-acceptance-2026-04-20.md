# DedupStudio Electron 验收计划（2026-04-20）

## 背景

浏览器态主流程和人物增强链路已经较稳定，但桌面端仍缺一轮与当前 web E2E 同等级别的复验闭环。

当前 Electron 已具备：

- 启动时检测并复用现有后端
- managed / external / error / stopped 状态透传
- 文件夹选择 IPC
- 主进程代理 API 调用

但仍缺少一份明确的桌面端复验清单，导致项目状态文件仍停在“转入桌面端复验”的旧阶段。

## 本轮验收目标

验证 Electron 开发态下，以下路径是否与浏览器态结论一致：

1. 启动应用
2. 确认后端状态展示
3. 选择目录
4. 扫描与分析
5. 打开对比面板
6. 标记移除
7. 执行移除
8. 撤销
9. 历史记录回显

## 重点观察项

### 后端来源

需要确认前端状态条能稳定区分：

- `内置后端已就绪`
- `已接管现有后端`
- `后端启动中`
- 异常态

### Dual + identity v2

需要使用包含以下场景的目录：

- 同人重复图
- 同人轻微姿态变化
- 同模板不同人
- 普通重复图

重点观察：

- `different person` 是否被真正排除
- `uncertain` 是否按 fallback 处理，而不是误判成强 same
- ComparePanel 中 Winner 和待删状态是否一致

### 撤销链路

需要确认：

- 执行移除后，目标目录生成 `-已去重`
- 撤销后文件能恢复
- 历史面板可见最近一次动作
- 重建目录后不会继承旧 `_undo_stack` / history 残留

## 推荐手测顺序

1. 启动 `npm run dev`
2. 打开 Electron 窗口
3. 观察首页后端状态标签
4. 通过菜单或按钮选择目录
5. 保持 `双保险` 策略
6. 点击 `开始分析`
7. 打开第一组对比图
8. 标记至少 1 张移除
9. 执行移除
10. 立刻执行撤销
11. 观察历史、统计数和磁盘文件恢复

## 输出要求

本轮验收完成后，建议补一份结果记录，至少包含：

- 运行模式：managed / external
- 测试目录
- 是否完成分析
- 是否能打开 ComparePanel
- 是否能移除
- 是否能撤销
- 是否发现桌面端独有问题

## 当前结论

截至 2026-04-20，本文件仍是验收计划，不代表 Electron 已完成等价验收。
