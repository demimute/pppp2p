# PPPP2P 用户手册

## 1. 这是什么

PPPP2P 是一个本地运行的照片去重工具，用来帮助你从一批相似、连拍、重复、聊天截图或轻微变体图片中，快速找出可以清理的内容。

它不是“自动替你删库”的工具，而是“先帮你分组，再由你确认”的工具。你始终可以查看候选组、指定保留项、再执行移除，因此更适合处理重要照片目录。

## 2. 能做什么

当前版本支持：

- 选择一个本地照片目录。
- 扫描该目录中的图片。
- 将重复或高相似图片整理成分组。
- 在组内逐张查看并对比。
- 指定要保留的图片。
- 执行移除，把待删项移出当前工作集。
- 撤销最近一次移除。
- 查看最近处理历史。

## 3. 运行环境

### macOS

- 可直接使用桌面应用构建产物。

### Windows

- 建议使用完整解压后的绿色版目录。
- 不要只复制 `exe` 单文件运行。

## 4. 启动方式

### 开发态启动

如果你是在源码环境下使用：

```bash
npm install
npm run dev
```

这会启动前端构建并拉起 Electron 桌面窗口。

### 打包版启动

如果你拿到的是打包产物：

- macOS：双击应用打开。
- Windows：完整解压后，双击 `PPPP2P` 或 `DedupStudio.exe` 启动。

如果首次启动被系统拦截，按系统提示选择继续运行即可。

## 5. 基本使用流程

### 第一步：选择照片目录

启动应用后，先选择你要分析的照片目录。

建议：

- 使用一个专门整理中的目录。
- 第一次使用时，先拿副本目录试跑。
- 不建议一上来就在不可恢复的重要原始目录上做大规模操作。

### 第二步：开始分析

选择目录后，点击开始分析。

系统会执行几类工作：

- 扫描图片列表
- 计算视觉 embedding
- 计算 hash
- 生成相似组
- 在需要时补充人物身份与姿态相关判断

如果目录较大，首次分析可能需要一点时间。之后再次分析通常会更快。

### 第三步：查看分组结果

分析完成后，你会看到候选分组列表。每一组代表一批系统认为可能重复或高度相似的图片。

分组不等于“系统已经替你决定删除”，它只是待你复核的候选集合。

### 第四步：进入对比面板

打开某一组后，可以逐张检查图片，确认：

- 哪张最清晰
- 哪张构图最好
- 哪张是你真正想保留的版本
- 哪些只是近似重复或较差副本

### 第五步：指定保留项

建议每组先确定一张保留图。

一般可以按这些标准决定：

- 清晰度更高
- 人脸/主体状态更好
- 构图更完整
- 没有裁切、水印或压缩痕迹
- 是原图而不是转发图、截图或缩略图

### 第六步：执行移除

确认后执行移除。系统会处理被标记的候选项，并记录本次动作，供后续撤销。

执行前建议你再看一遍待移除数量，避免误操作。

### 第七步：需要时执行撤销

如果刚才的移除结果不满意，或者发现误删，可以立刻使用撤销功能恢复最近一次操作。

撤销是当前工作流里非常重要的一环。你应当把它当作正常能力，而不是“出错时的最后补救”。

## 6. 使用建议

### 6.1 第一次先用小目录试跑

不要一开始就拿几万张照片做全量尝试。先用一个几十到几百张的目录，确认你理解界面和分组逻辑，再扩大范围。

### 6.2 优先看分组，不要盲信分数

系统分组已经很有参考价值，但最终保留哪张，还是应该以你的判断为准。

### 6.3 重要目录先做副本

如果这批照片对你很重要，建议先复制出一个测试目录，再做第一次正式清理。

### 6.4 每次处理一批，不要一次性做太大

比起一次处理整个总图库，更推荐按相册、日期、来源目录逐批处理。这样更稳，也更容易复核。

## 7. 它是怎么判断相似的

你不需要理解所有实现细节，但理解大致原则会更容易用好它。

PPPP2P 当前主要参考以下信号：

- 整体视觉相似度
- 图像结构相似度
- 人物身份相关信号
- 同一人物前提下的姿态差异

简单说：

- 先判断两张图整体像不像。
- 再判断结构是不是接近。
- 如果图里有人，还会额外参考是不是同一个人。
- 只有在确认同人的前提下，姿态相近度才会参与细化。

这能减少“同模板但不同人”被误并的概率。

## 8. 常见问题

### 8.1 为什么有些明显相似的图没有进同一组？

可能原因：

- 两张图结构差异太大。
- 一张是裁切版，一张是大幅改色版。
- 人物或主体差异让系统更保守。
- 当前阈值更偏向降低误并。

如果你的目标是“宁可多召回，再人工筛”，后续可以考虑继续优化策略参数。

### 8.2 为什么有些组里混进了不该在一起的图？

这通常是相似图工具不可完全避免的问题，尤其在：

- 相同模板截图
- 连续动作很近但主体不同
- 光线/背景强相似
- 无脸或小主体场景

当前版本已经比早期版本更重视“异人误并”的抑制，但不是所有边界情况都已经彻底解决。

### 8.3 移除后文件去哪了？

这取决于当前实现的处理方式与输出目录策略。你在执行移除前，最好先观察界面提示或在测试目录中先跑一遍，确认输出目录命名和落盘行为。

如果你刚执行过移除且结果不符合预期，优先使用撤销功能。

### 8.4 为什么首次启动比较慢？

首次启动可能会涉及：

- Electron 启动
- 本地后端拉起
- 模型或缓存准备
- 首次读取目录和计算特征

这属于正常现象。后续再次使用通常会更快。

### 8.5 Windows 下打不开怎么办？

先检查以下几点：

- 是否完整解压了整个目录
- 是否只双击了单独的 `exe`
- 是否被系统安全策略或杀毒软件拦截
- 是否下载了错误架构的产物

当前 Windows 打包链路仍是项目的重点观察项之一，因此如果遇到“能启动壳但后端不可用”的问题，不一定是你操作错了，也可能是当前产物本身有缺陷。

## 9. 当前版本的边界

当前版本已经可用，但你应该知道它的边界：

- 它更适合“辅助筛选 + 人工确认”，不是全自动删除系统。
- Electron 桌面端已经可运行，但桌面 UI 自动化回归还不算充分。
- Windows 自包含分发链路近期仍发现过问题。
- 人物增强对很多场景有效，但还没有做到对所有复杂人物场景都稳。

## 10. 更安全的使用方式

如果你要把它用于真实照片库，推荐这个顺序：

1. 先复制一个测试目录。
2. 在测试目录里完整跑一遍分析、移除、撤销。
3. 确认自己理解每个按钮和输出结果。
4. 再逐批处理正式照片目录。
5. 每批处理后检查结果，不要长时间连续无复核地执行大量移除。
